# 变量遮蔽一个非常关键且有趣的特点，它和“作用域”（Scope）紧密相关

```rust
fn main() {
    // 这好比在“操场”上（外层作用域）
    let player = "一个站着不动的人"; 
    println!("在屋外，player 是: {}", player);

    { // <-- 我们现在走进了一个“没有窗户的屋子”（内层作用域）
        
        // 在这个屋子里，我们创建了一个新的、同名的“影子”变量
        let player = "player 的影子 (类型可以是全新的)"; 
        println!("在屋里，player 是: {}", player);

        // 在这个屋子里，任何地方提到 player，指的都是这个“影子”

    } // <-- 我们走出了屋子，内层作用域在这里结束。“影子”变量的生命也到此为止，它消失了！

    // 回到了外面的“操场”
    println!("回到屋外，player 是: {}", player); // 这里引用的又是最初的那个 player
}
```


# 电商网站的后台

```rust
fn process_order() {
    // 1. 从前端接收到一个字符串 " 3 "，注意前后有空格
    let mut quantity_str = " 3 ";

    // 2. 使用变量遮蔽去除空格
    //    这里创建了一个新的 quantity_str 变量，它是旧变量去除空格后的版本
    // 我换成了 mut
    quantity_str = quantity_str.trim(); // .trim() 会去掉字符串首尾的空白字符

    // 3. 再次使用变量遮蔽，将字符串转换为数字
    //    这里我们创建了一个全新的 quantity_num 变量，它的类型是 u32 (无符号32位整数)
    //    .parse() 方法尝试将字符串解析成某种数字类型
    //    .expect() 会在转换失败时让程序崩溃并显示错误信息，这里我们假设输入总是有效的
    let quantity_num: u32 = quantity_str.parse().expect("请输入一个有效的数字！");
    
    // 现在我们可以安全地使用这个数字进行计算了
    if quantity_num > 0 {
        println!("订单处理中：数量为 {} 件。", quantity_num);
        // 接下来可以进行减库存、计算总价等操作...
    }
}
```
